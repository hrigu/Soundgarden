# 3 - Feature (Spatial-Audio): Binaurales Insekt-Prototyp

## 1. Why

### 1.1 Objective

Ein erstes bewegliches Klangobjekt ("Insekt") bauen, das nach einer Bewegungsregel durch
einen definierten Raum kurvt und dem Hörer über Kopfhörer binaural zugespielt wird — mal
vorne, mal hinten, mal laut (nah), mal leise (fern).

### 1.2 Context

Ursprüngliche Idee des Nutzers für eine räumliche Klanginstallation. Erster Prototyp mit
genau einem Objekt und klassischer OOP-Struktur, wie vom Nutzer vorgegeben (`Movable`,
`Listener`), bewusst ohne Schwarmverwaltung/Mehrfach-Objekte für diesen ersten Schritt.

### 1.4 Related to Intents

* 4 - Feature (Spatial-Audio): Echtes HRTF für das Insekt — nächste Ausbaustufe der hier
  bewusst einfach gehaltenen Binauralisierung.

## 2.0 What

```gherkin
Feature: Binaurales Insekt

  Als Hörer mit Kopfhörern
  möchte ich ein virtuelles Klangobjekt räumlich wahrnehmen können
  damit es sich anfühlt, als würde ein Insekt tatsächlich um mich herumfliegen.

  Scenario: Insekt bewegt sich auf fester Bahn
    Given ein Movable mit einer Kreisbahn-Bewegungsregel
    When step() wiederholt aufgerufen wird
    Then ändert sich seine Position auf einer atmenden Kreisbahn um den Hörer

  Scenario: Position wird binaural hörbar
    Given die Position des Insekts relativ zum Listener
    When sie sich ändert
    Then hört man es über Pan + winzige Laufzeitdifferenz zwischen den Ohren lauter/leiser
      und heller/dumpfer, je nach Azimuth und Distanz

  Scenario: Insekt bleibt innerhalb der Raumkugel
    Given eine definierte Raumkugel (roomRadius)
    When die Bewegungsregel das Insekt darüber hinaus bewegen würde
    Then steuert Movable sanft zurück, statt hart abzuprallen
```

## 3.0 How

### 3.1 Implementation Context

* `classes/Movable.sc` — Position + Bewegungsregel (Function) + sanfte Grenzsteuerung
  innerhalb einer Raumkugel.
* `classes/Listener.sc` — aktuell fixe Position/Blickrichtung; berechnet aus einer
  Weltposition Azimuth (relativ zur eigenen Blickrichtung) und Distanz.
* `classes/SoundInsect.sc` — verbindet ein `Movable` mit einem laufenden `Synth`; ein
  `Routine` tickt die Bewegung und schiebt Azimuth/Distanz live in den Synth.
* `insect_demo.scd` — `SynthDef \insectVoice`: Gleichleistungs-Pan (`Pan2`) + winzige
  Laufzeitdifferenz zwischen den Ohren (`DelayL`) + entfernungsabhängiger Pegel/Tiefpass.
  Reine Core-UGens, kein echtes HRTF — bewusste erste, einfache Näherung.
