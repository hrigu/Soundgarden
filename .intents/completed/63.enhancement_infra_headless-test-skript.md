# 63 - Enhancement (Infra): Headless-Test-Skript aus AGENTS.md auslagern

## 1. Why

### 1.1 Objective
Das in `AGENTS.md` beschriebene Rezept für headless Testläufe ist detailliert, aber
fehleranfällig: jeder Agent muss es bei jeder Ausführung aus dem Fließtext rekonstruieren
(`run_tests.scd` kopieren, `0.exit;` anhängen, Hintergrund-Monitoring, etc.). Ein kleines
Helfer-Skript reduziert diesen Aufwand und das Risiko hängender `sclang`-Prozesse.

### 1.2 Context
`AGENTS.md` enthält seit Intent 6 bzw. späteren Erweiterungen den Abschnitt
"Headless-Testlauf vor dem Push" mit dem genauen Rezept. Dieses Rezept funktioniert, ist
aber für wiederholte Ausführung unhandlich.

### 1.3 Related To Intents
- 6 - Tests von Produktivcode trennen
- 32 - load-classes Doku pro Sitzung

## 2.0 What

```gherkin
Feature: Headless-Test-Skript

  Scenario: Tests über ein Skript ausführen
    Given ich möchte alle Unit-Tests headless laufen lassen
    When ich `scripts/run_tests.sh` ausführe
    Then kopiert das Skript `run_tests.scd` temporär
    And hängt `0.exit;` an
    And startet `/Applications/SuperCollider.app/Contents/MacOS/sclang`
    And beendet sich sauber mit dem Exit-Code von sclang

  Scenario: Dokumentation zeigt auf das Skript
    Given `AGENTS.md` beschreibt den Headless-Testlauf
    When ein Agent die Anleitung liest
    Then verweist sie auf `scripts/run_tests.sh` statt das komplette Rezept zu wiederholen
```

## 3.0 How

### 3.1 Implementation Context
- Neues Skript `scripts/run_tests.sh` im Repo-Root.
- `AGENTS.md`: Headless-Test-Abschnitt kürzen und auf das Skript verweisen, wichtige
  Hinweise (kein Pipe durch `tail`/`head`, voller `sclang`-Pfad) beibehalten.
