# 14 - Enhancement (Infra): sg-Namespace als Ordnerstruktur einführen

## 1. Why

### 1.1 Objective

Die Spatial-Audio-Klassen unter einem gemeinsamen `sg`-Namespace organisieren, darunter
gegliedert in drei sinnvolle Unter-Namespaces: `sounds`, `soundobjects`, `spatial`.

### 1.2 Context

`classes/` sammelt bisher alle Klassen flach in einem Ordner. Mit wachsender Klassenzahl
(Intent 9–12: `Movable`, `MoveRule`, `CircularMoveRule`, `Sound`, `InsectSound`, `SoundObject`,
`Listener`, `Binauralizer`, `Orchestra`) wird das unübersichtlich. SuperCollider kennt aber
keine echten Namespaces (ein globaler, flacher Klassen-Namensraum, keine Punktnotation in
Klassennamen) — der Nutzer hat sich nach Abwägung für eine reine Ordnerstruktur entschieden
(kein Klassennamen-Präfix), da das risikoärmer ist: keine Referenz im Code/Tests/Doku ändert
sich, nur der Dateipfad.

`BootTrackDetection` (Live-Set-Domäne, Track-Erkennung fürs DJ-Rig) gehört bewusst **nicht**
in `sg` — andere Domäne, bleibt direkt in `classes/`.

### 1.3 Depends On Intents

* 9, 10, 11, 12 - lieferten die Klassen, die hier umsortiert werden.

## 2.0 What

```gherkin
Feature: sg-Namespace als Ordnerstruktur

  Als Entwickler
  möchte ich die Spatial-Audio-Klassen thematisch in Unterordnern finden
  damit sich die Klassenbibliothek trotz wachsender Zahl an Klassen übersichtlich anfühlt.

  Scenario: Klassen sind thematisch gruppiert
    Given classes/sg/{sounds,soundobjects,spatial}/
    When ich eine Klasse suche (z.B. "wo ist die Klangerzeugung?")
    Then finde ich sie im passenden Unterordner

  Scenario: Alles kompiliert weiterhin unverändert
    Given load_classes.scd bindet classes/ und tests/ weiterhin rekursiv ein
    When ich load_classes.scd ausführe
    Then kompiliert die Klassenbibliothek ohne Fehler, alle Klassennamen unverändert

  Scenario: Bestehende Tests bleiben grün
    Given die Testsuite lief vor dem Verschieben grün
    When ich run_tests.scd (bzw. den headless sclang-Lauf) danach ausführe
    Then sind alle Tests weiterhin grün, ohne Codeänderung an ihrer Logik
```

## 3.0 How

### 3.1 Implementation Context

`LanguageConfig.addIncludePath` scannt rekursiv — Unterordner unter `classes/`/`tests/` werden
automatisch mitkompiliert, `load_classes.scd` selbst muss nicht angepasst werden. Reine
Dateiverschiebung (`git mv`), keine Klasseninhalte ändern sich.

Zielstruktur:
```
classes/
  BootTrackDetection.sc          (bleibt, andere Domäne)
  sg/
    sounds/         Sound.sc, InsectSound.sc
    soundobjects/   SoundObject.sc, Movable.sc, MoveRule.sc, CircularMoveRule.sc
    spatial/        Listener.sc, Binauralizer.sc, Orchestra.sc
tests/
  TestBootTrackDetection.sc      (bleibt)
  sg/
    soundobjects/   TestCircularMoveRule.sc
    spatial/        TestOrchestra.sc
```
