# 1 - Feature (Live-Set): Live-Coding-Rig Grundgerüst

## 1. Why

### 1.1 Objective

Ein Live-Coding-Setup in SuperCollider aufbauen, mit dem bestehende elektronische Tracks
während einer Fusion-Tanzparty in Echtzeit umgeformt werden können (Filter, Echo,
Stutter/Glitch, Bitcrush, Reverb) — ohne Aussetzer, rein über Tastatur.

### 1.2 Context

Ziel ist eine Live-Performance ohne MIDI-Controller (nur Tastatur, `Cmd+Enter` je Block).
JITLib (`Ndef`) ermöglicht das klickfreie Neudefinieren laufender Synths — genau das macht
SuperCollider für diesen Zweck geeignet.

## 2.0 What

```gherkin
Feature: Live-Coding-Rig Grundgerüst

  Als DJ
  möchte ich bestehende Tracks live durch Filter/Echo/Bitcrush/Stutter/Reverb schicken können
  damit ich die Tanzmenge während eines Fusion-Sets in Echtzeit mitnehmen kann.

  Scenario: Server bootet und lädt Tracks automatisch
    Given Audiodateien liegen in sounds/
    When ich boot.scd ausführe
    Then werden alle unterstützten Dateien (.wav/.aiff/.aif/.flac) als Buffer in ~tracks geladen

  Scenario: FX-Kette live umschalten
    Given ein Track läuft über Ndef(\track)
    When ich Ndef(\fx) mit einer anderen FX-Kette aus ~fxChains neu definiere
    Then ändert sich der Klang ohne Klicken oder Aussetzer

  Scenario: Track wechseln ohne Unterbrechung
    Given ein Track läuft
    When ich Ndef(\track) mit einem anderen Buffer neu definiere
    Then überblendet fadeTime automatisch zum neuen Track
```

## 3.0 How

### 3.1 Implementation Context

* sclang + scsynth (Client/Server), JITLib (`Ndef`) für klickfreies Live-Editing.
* `boot.scd` bootet den Server und scannt `sounds/` automatisch in `~tracks`.
* `fx.scd` definiert `~fxChains`: `clean`, `filterSweep` (RLPF), `echo` (CombN),
  `bitcrush` (Latch + round), `stutterGate` (LFPulse), `reverb` (FreeVerb).
* `set_template.scd` als Performance-Vorlage mit Beispiel-Blöcken für
  Build-up/Drop/Break/Übergang.

## 4. Challenges & Solutions

* `PathName>>extension` lieferte teils ein Symbol statt einen String zurück;
  `Array:-includes`/`-indexOf` vergleichen intern per Identität, wodurch der
  Extension-Check trotz sichtbar identischem Text fehlschlug. Gelöst durch Vergleich
  als Symbol statt String.
* `Decimator` (für Bitcrush) gehört zum optionalen sc3-plugins-Paket, nicht zum Core —
  `ERROR: Class not defined` auf frischer Installation. Gelöst durch Nachbau mit
  Core-UGens (`Latch` + `round`).
