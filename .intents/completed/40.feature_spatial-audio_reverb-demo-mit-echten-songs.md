# 40 - Feature (Spatial-Audio): Reverb-Demo mit echten Songs aus musicnet/downloads

## 1. Why

### 1.1 Objective
Ein neues Demo unter `demos/reverb/` bereitstellen, das echte Songs aus
`/Users/chrigu/code/my/musicnet/downloads` in einem `Room` abspielt, damit der Raumhall
nicht nur mit kurzen Testklängen, sondern mit vollständigem Musikmaterial beurteilt werden kann.

### 1.2 Context
Die bisherigen Reverb-Demos arbeiten mit:

* einzelnen Samples
* synthetischen Insekten
* analytischen Kurzsequenzen

Für Hall-Beurteilung mit realem Material fehlt noch ein Setup mit echten Songs. Das soll:

* denselben `Room`- und `SpatialControlPanel`-Workflow nutzen
* eine Song-Auswahl per Name erlauben
* einen festen, nicht bewegten Klangursprung verwenden
* Live-Tuning der Room-Parameter im gemeinsamen GUI erlauben

### 1.3 Related To Intents

* 24 - Reverb-Testsetup einzelner Klang
* 35 - Raumhall-Analyse-Demo
* 39 - Steuerung, Raum-Draufsicht und Room-GUI vereinen

## 2.0 What

```gherkin
Feature: Reverb-Demo mit echten Songs

  Scenario: Song per Name aus musicnet/downloads laden
    Given ich setze eine Environment-Variable mit einem Song-Namen oder Namensfragment
    When ich das Demo-Setup ausführe
    Then wird eine passende Audiodatei aus /Users/chrigu/code/my/musicnet/downloads gewählt

  Scenario: Song in festem Raum-Setup abspielen
    Given ein geladener Song
    When ich das Demo starte
    Then höre ich den Song als stationäres SoundObject im Room
    And ich kann den Listener im gemeinsamen GUI bewegen

  Scenario: Room-Parameter live verändern
    Given das gemeinsame SpatialControlPanel ist offen
    When ich size, height, surface, mix oder reverbMix ändere
    Then verändert sich der Hall des laufenden Songs live

  Scenario: Demo sauber stoppen und neu starten
    Given das Demo lief bereits
    When ich stoppe und erneut starte
    Then läuft es ohne zusätzliches Neuverdrahten wieder an
```

## 3.0 How

### 3.1 Implementation Context
Neues Skript:

* `demos/reverb/reverb_song.scd`

Geplantes Verhalten:

* rekursive Dateisuche in `/Users/chrigu/code/my/musicnet/downloads`
* Auswahl über `~songName`:
  * exakter Match auf `fileNameWithoutExtension` bevorzugt
  * sonst Namensfragment
  * ohne Vorgabe: erste gefundene Datei
* ein stationäres `SampleSound`-basiertes `SoundObject`
* `Room.new(...)` + `AtkBinauralizer`
* gemeinsames `SpatialControlPanel`
* komfortable Helfer wie `~startDemo`, `~showControls`, `~stopDemo`, `~freeDemo`
