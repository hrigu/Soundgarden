# 34 - Enhancement (Spatial-Audio): SCDoc-Hilfeseiten für Room/RoomReverb (Hall-Parameter)

## 1. Why

Beim Herumspielen mit `demos/grill/insects.scd` fehlt eine nachschlagbare Doku dafür, wie man
z.B. den Hall einstellt (`~room.size`/`.height`/`.surface`/`.mix`). SuperCollider bietet dafür
sein eingebautes SCDoc-Hilfesystem: `.schelp`-Dateien in einem `HelpSource`-Ordner werden
automatisch erkannt, sobald der übergeordnete Ordner als `LanguageConfig.includePath`
registriert ist (experimentell verifiziert). Damit landet die Doku für `Room`/`RoomReverb`
direkt im normalen HelpBrowser, durchsuchbar und per `cmd-d` auf den Klassennamen aufrufbar —
statt in einer separaten Markdown-Datei.

## 2.0 What

```gherkin
Given HelpSource/Classes/Room.schelp beschreibt Konstruktor, die Hall-Parameter
  (size/height/surface/mix) und die Kernmethoden von Room
And HelpSource/Classes/RoomReverb.schelp beschreibt kurz die interne Rolle von
  RoomReverb mit Verweis auf Room
And die SCDoc-Dateien liegen direkt im Projekt-HelpSource-Ordner
When jemand in scide einmalig Help → Rebuild All Documentation ausführt und danach den
  Cursor auf "Room" setzt und cmd-d drückt
Then öffnet sich die eigene Room-Hilfeseite im normalen HelpBrowser, mit Erklärung von
  size/height/surface/mix

Given ein Entwickler möchte künftig weitere sg-Klassen dokumentieren
When er eine weitere .schelp-Datei unter HelpSource/Classes/ ablegt
Then wird sie automatisch mit erfasst, ohne weitere Konfigurationsänderungen
```

## 3.0 How

Implementation Context: neuer Ordner `HelpSource/Classes/` mit `Room.schelp` und
`RoomReverb.schelp` (SCDoc-Markup: `class::`, `summary::`, `categories::`, `description::`,
`instancemethods::`/`classmethods::` mit `method::`/`argument::`, `examples::` z.B.
`~room.size = 12;`). Der zuerst geplante Ablageort `docs/HelpSource/` wurde verworfen:
experimentell verifiziert funktioniert die eigene Hilfe zuverlässig, wenn `HelpSource/`
direkt im Projektverzeichnis liegt.

- [x] 1.0 `boot/load_classes.scd`: Kommentar zum projektweiten `HelpSource/` ergänzen
- [x] 2.0 `HelpSource/Classes/Room.schelp` schreiben (Konstruktor, `addSynthDef`,
      `register`/`registerFromBuilder`, `call`, `play`/`stop`, live-Setter
      `size`/`height`/`surface`/`mix` mit Beispiel)
- [x] 3.0 `HelpSource/Classes/RoomReverb.schelp` schreiben (kurz: interne Rolle,
      Verweis auf `Room` für die eigentliche Bedienung)
- [x] 4.0 Verifikation durch den Nutzer: Help → Rebuild All Documentation, `cmd-d` auf `Room`
      testen, Befund bestätigen
- [x] 5.0 `README.md`: kurzer Hinweis, dass eigene Klassen jetzt SCDoc-Hilfeseiten haben
      (Help → Rebuild All Documentation nötig nach Class-Library-Änderungen)

## Challenges & Solutions

- `docs/HelpSource/` wurde nicht zuverlässig vom HelpBrowser gefunden. Der Ordner liegt deshalb
  direkt im Projektverzeichnis als `HelpSource/`; nach `Help → Rebuild All Documentation` ist
  `Room` per `cmd-d` aufrufbar.
