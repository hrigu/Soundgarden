# 44 - Enhancement (Spatial-Audio): Einfaches RoomControlPanel als Demo-Wrapper

## 1. Why

### 1.1 Objective
Ein sehr einfaches, wiederverwendbares GUI-API für Demos bereitstellen, damit ein
Room-Control-Panel mit wenigen Zeilen eingebunden werden kann, ohne jedes Demo direkt an
`SpatialControlPanel` zu koppeln.

### 1.2 Context
Aktuell ist `SpatialControlPanel` die mächtige, vollständige Oberfläche:

* Raum-Draufsicht
* Listener-Steuerung
* Room-/Hall-Regler
* Soundobjekt-Auswahl
* optionale Presets

Für viele Demos ist das mehr als nötig. Gewünscht ist ein bewusst kleines API, ungefähr so:

* `panel = RoomControlPanel.new(room)`
* `panel.show()`
* `panel.hide()`
* `panel.free()`

Das soll die bestehende Funktionalität nicht ersetzen, sondern als einfache Hülle für
Demo-Skripte bereitstellen.

### 1.3 Related To Intents

* 39 - Steuerung, Raum-Draufsicht und Room-GUI vereinen
* 41 - GVerb-Parameter als Room-Eigenschaften im GUI
* 43 - live editierbare Grillen-Sounds mit Preset-Bibliothek

## 2.0 What

```gherkin
Feature: Einfaches RoomControlPanel für Demo-Skripte

  Scenario: Room-Control-Panel mit einfacher API öffnen
    Given ein bestehender Room
    When ich RoomControlPanel.new(room) erzeuge und show aufrufe
    Then erscheint das Control-Panel für diesen Room

  Scenario: Panel vorübergehend ausblenden
    Given ein sichtbares RoomControlPanel
    When ich hide aufrufe
    Then ist das Fenster verborgen, aber nicht freigegeben

  Scenario: Panel wieder anzeigen
    Given ein zuvor verborgenes RoomControlPanel
    When ich erneut show aufrufe
    Then erscheint dasselbe Fenster wieder

  Scenario: Panel vollständig freigeben
    Given ein bestehendes RoomControlPanel
    When ich free aufrufe
    Then werden Fenster und interne GUI-Ressourcen sauber freigegeben

  Scenario: Demos können das API als Dreizeiler nutzen
    Given ein Demo mit Room
    When es das RoomControlPanel einbindet
    Then braucht es nur new/show/hide/free statt direkter SpatialControlPanel-Details
```

## 3.0 How

### 3.1 Implementation Context

* Neue Klasse: `classes/sg/gui/RoomControlPanel.sc`
* `RoomControlPanel` ist ein dünner Wrapper um `SpatialControlPanel`, keine zweite unabhängige
  GUI
* Festgelegte API:
  * `*new { |room, viewRadius = 8, moveSpeed = 2, rotateSpeed = 90, editable = true,
    presetsDir| ... }`
  * `show` — erzeugt das innere `SpatialControlPanel` lazy beim ersten Aufruf, zeigt danach
    immer dasselbe Fenster wieder an
  * `hide` — blendet das Fenster nur aus, ohne das innere Panel freizugeben
  * `free` — beendet/freigibt das innere Panel vollständig; ein späteres `show` baut es neu auf
* `RoomControlPanel` hält intern genau eine `SpatialControlPanel`-Instanz und reicht die
  Konstruktor-Parameter unverändert an sie weiter
* Optionale Default-Parameter bleiben intern möglich (`viewRadius`, `editable`, `presetsDir`),
  aber der Standardfall bleibt `RoomControlPanel.new(room)`
* Demo-Zielbild:

  ```supercollider
  ~panel = RoomControlPanel.new(~room);
  ~panel.show();
  ~panel.hide();
  ~panel.free();
  ```
