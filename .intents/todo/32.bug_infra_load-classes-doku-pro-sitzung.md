# 32 - Bug (Infra): Doku beschreibt load_classes.scd fälschlich als Pro-Sitzung-Pflichtschritt

## 1. Why

`LanguageConfig.addIncludePath` persistiert standardmäßig in `sclang_conf.yaml`. Diese Datei
enthält bereits die Pfade zu `classes/` und `tests/` — vermutlich seit dem ersten Lauf von
`load_classes.scd`. Damit kompiliert SuperCollider diese Klassen ab jetzt bei **jedem** Start
automatisch mit, unabhängig von `load_classes.scd`. Die Doku (`CLAUDE.md`, `AGENTS.md`,
`README.md`, Kommentare in `run_tests.scd` und diversen Demos) beschreibt den Schritt aber
weiterhin als "einmal pro Sitzung" nötig — unnötiger manueller Aufwand.

## 2.0 What

```gherkin
Given sclang_conf.yaml enthält bereits die persistenten includePaths für classes/ und tests/
When SuperCollider neu gestartet wird, ohne load_classes.scd manuell auszuführen
Then sind Klassen aus classes/ und tests/ sofort verfügbar

Given die aktualisierte Doku
When jemand das Repo liest
Then wird klar, dass load_classes.scd nur beim Erst-Setup bzw. nach einem Reset von
  sclang_conf.yaml nötig ist, nicht bei jedem Start
```

## 3.0 How

- [ ] 1.0 Verifikation durch den Nutzer: SuperCollider neu starten, `demos/grill/insects.scd`
      **ohne** vorheriges `load_classes.scd` testen, Befund bestätigen
- [ ] 2.0 Doku anpassen: `CLAUDE.md`, `AGENTS.md`, `README.md`, `run_tests.scd`-Kommentar,
      Demo-Kommentare (`insects.scd`, `one_insect.scd`, `insectsound.scd`, `sample.scd`,
      `sample_moving.scd`, `reverb.scd`), `experiments/live_coding_rig/README.md`
- [ ] 3.0 Kommentar in `boot/load_classes.scd` anpassen (nur Erst-Setup/Reset, nicht pro
      Sitzung)
