# 37 - Enhancement (Spatial-Audio): Room optional ohne Hall konfigurierbar

## 1. Why

### 1.1 Objective
`Room` so erweitern, dass er optional ganz ohne Hall betrieben werden kann, ohne dass Demo-
Skripte dafür globale Parameter wie `mix = 0` und `reverbMix = 0` an mehreren Stellen
gleichzeitig setzen müssen.

### 1.2 Context
Aktuell ist Hall immer Teil von `Room`: `RoomReverb` wird konstruiert, `Room>>play` startet
den Reverb-Synth, und `register`/`registerFromBuilder` geben jedem Binauralizer standardmäßig
einen Reverb-Send mit. Für A/B-Hörtests ist das unnötig umständlich: „trocken“ bedeutet heute
mehrere Parameter manuell auf `0` zu setzen, statt den Raum explizit als „ohne Hall“ zu
konfigurieren.

Ein expliziter Schalter macht den Zustand fachlich klarer:

* trockener Referenzraum für Ortungs-/Klangvergleich
* gleicher `Room`-/`Listener`-/`Orchestra`-Workflow wie mit Hall
* keine implizite Abhängigkeit davon, dass `mix` und `reverbMix` zufällig beide auf `0` stehen

### 1.3 Related To Intents

* 26 - Room-Parameter steuern Hall
* 35 - Raumhall-Analyse-Demo mit stationären Zirp- und Sample-Objekten
* 36 - Trockeneren Room/Hall als neue Defaults setzen

## 2.0 What

```gherkin
Feature: Room optional ohne Hall

  Scenario: Room wird trocken konfiguriert
    Given ich erzeuge einen Room mit explizit deaktiviertem Hall
    When ich ihn abspiele
    Then startet kein Reverb-Synth
    And registrierte Soundobjekte bleiben trotzdem hörbar
    And der restliche Room-Workflow bleibt unverändert

  Scenario: Bestehender Room mit Hall bleibt Standard
    Given ich erzeuge einen Room ohne zusätzlichen Schalter
    When ich ihn wie bisher benutze
    Then bleibt das bestehende Verhalten mit Hall erhalten
```

## 3.0 How

### 3.1 Implementation Context
Voraussichtlich:

* `classes/sg/spatial/Room.sc`: optionales Flag wie `reverbEnabled = true`
* `Room>>play`: startet `RoomReverb` nur, wenn Hall aktiviert ist
* `Room>>register` / `registerFromBuilder`: bei deaktiviertem Hall `reverbMix: 0` bzw.
  kein Hall-Send in den Binauralizer
* `tests/sg/spatial/TestRoom.sc`: trockenes Verhalten absichern
* `HelpSource/Classes/Room.schelp`: Schalter dokumentieren
* optional `demos/reverb/reverb_analysis.scd`: einfacher A/B-Schalter im `config`
