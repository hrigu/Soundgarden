# 69 - Feature (Spatial-Audio): Rhythm-Pattern-Playground für isoliertes Testen und Kombinieren

## 1. Why

### 1.1 Objective
Ein kleiner, schneller Playground, in dem die Rhythm-Patterns aus `cricket_techno.scd`
einzeln und frei kombiniert getestet werden können — ohne den vollen 36-Stimmen-Raum
starten zu müssen. Damit können die Patterns besser gehört, verglichen und verstanden
werden, bevor sie ins Hauptstück übernommen werden.

### 1.2 Context
In `cricket_techno.scd` laufen 8 Schichten gleichzeitig. Um zu verstehen, was Kick,
Snare, Clap oder Hihat jeweils rhythmisch beitragen, braucht es einen einfacheren
Testraum mit wenigen Stimmen und schnellem Start/Stop.

### 1.3 Related To Intents
- 68 - Art-Techno-Grillen-Experiment mit 36 Stimmen und 16-Takt-Pattern

## 2.0 What

```gherkin
Feature: Rhythm-Pattern-Playground

  Scenario: Einzelnes Pattern isoliert abspielen
    Given der Playground läuft
    When der Benutzer nur ein Layer aktiviert
    Then hört er genau dieses Pattern auf einem einfachen CricketSound

  Scenario: Mehrere Layer kombinieren
    Given der Playground läuft
    When der Benutzer mehrere Layer einschaltet
    Then werden deren Patterns gemeinsam abgespielt

  Scenario: Pattern editieren und sofort hören
    Given der Benutzer ändert einen Pattern-String
    When er den Playground neu startet
    Then ändert sich der Rhythmus sofort
```

## 3.0 How

### 3.1 Implementation Context
- Neues Skript unter `experiments/cricket_techno/rhythm_playground.scd`.
- Nutzt bestehende Klassen: `CricketSound`, optional `Room`/`Listener` für binaurale
  Wiedergabe.
- Die Pattern-Definitionen aus `cricket_techno.scd` werden wiederverwendet oder in eine
  gemeinsame Hilfsdatei ausgelagert.
