# 33 - Feature (Spatial-Audio): RhythmPatternCreator mit konfigurierbarem Sound/Stille-Verhältnis

## 1. Why

`~makeRhythmPattern` in `demos/grill/insects.scd` erzeugt aktuell An/Aus-Segmente mit
gleichverteilt gewichtetem Zufall, unabhängig davon ob ein Segment An oder Aus ist — weder das
Sound/Stille-Verhältnis noch die Startphase sind steuerbar. `RhythmPatternCreator` macht beides
explizit konfigurierbar.

## 2.0 What

```gherkin
Given RhythmPatternCreator.new(totalDur: 3, numSegments: 12) (soundRatio: 0.5, startWithSound: true als Defaults)
When createNewPattern aufgerufen wird
Then ist die Summe der "An"-Segmentdauern ungefähr gleich der Summe der "Aus"-Segmentdauern,
  die Gesamtdauer aller Segmente ergibt exakt totalDur, und das erste Segment ist "An"

Given RhythmPatternCreator.new(totalDur: 3, numSegments: 12, soundRatio: 0.1)
When createNewPattern aufgerufen wird
Then macht die Summe der "An"-Segmentdauern ca. 10% von totalDur aus, die "Aus"-Segmente
  entsprechend ca. 90% — innerhalb der An- bzw. Aus-Gruppe bleiben die Einzeldauern weiterhin
  zufällig gewichtet

Given RhythmPatternCreator.new(totalDur: 3, numSegments: 12, startWithSound: false)
When createNewPattern aufgerufen wird
Then ist das erste Segment "Aus" (Stille), danach wechselt es weiterhin strikt ab

Given ein RhythmPatternCreator
When createNewPattern mehrfach aufgerufen wird
Then liefert jeder Aufruf eine neue, unabhängige CallingPattern-Instanz (kein geteilter Zustand)
```

## 3.0 How

Implementation Context: Neue Klasse `classes/sg/sounds/RhythmPatternCreator.sc`, Tests unter
`tests/sg/sounds/TestRhythmPatternCreator.sc`. Konstruktor ausschliesslich mit Named Parameters:
`RhythmPatternCreator.new(totalDur:, numSegments:, soundRatio: 0.5, startWithSound: true)`.
`createNewPattern` (keine Argumente) liefert ein `CallingPattern`. Intern: `numSegments`
gewichtete Zufallswerte erzeugen (`rrand(0.01, 2.0)`), nach Segment-Index in zwei Gruppen teilen
(Index `i`: An-Gruppe wenn `i.even == startWithSound`, sonst Aus-Gruppe), An-Gruppe auf
`soundRatio * totalDur` skalieren, Aus-Gruppe auf `(1 - soundRatio) * totalDur`, Reihenfolge
beibehalten, als `[dauer, istAn]`-Paare direkt an `CallingPattern.new` übergeben (nicht
`.fromDurations`, da dessen feste Start-mit-An-Konvention hier nicht passt).
