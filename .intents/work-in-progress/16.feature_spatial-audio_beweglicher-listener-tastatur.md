# 16 - Feature (Spatial-Audio): Beweglicher Listener per Tastatur

## 1. Why

### 1.1 Objective

Der Listener soll sich per Tastatur bewegen (vor/zurück/seitlich) und drehen können,
kontinuierlich solange eine Taste gehalten wird.

### 1.2 Context

Aktuell ist der Listener fix im Ursprung mit fixer Blickrichtung. Für eine überzeugendere
Live-Performance soll der Hörer sich im virtuellen Raum bewegen und umschauen können, während
Soundobjekte spielen — kontinuierliche, WASD-artige Steuerung über ein fokussiertes
Kontrollfenster (SuperCollider-Standardidiom für Tastatur-Input).

### 1.3 Depends On Intents

* 3 - Feature (Spatial-Audio): Binaurales Insekt-Prototyp — liefert `Listener`.

## 2.0 What

```gherkin
Feature: Beweglicher Listener

  Als Performer
  möchte ich mich mit der Tastatur durch den virtuellen Raum bewegen und drehen können
  damit Soundobjekte je nach meiner Position/Blickrichtung unterschiedlich klingen.

  Scenario: Vorwärts bewegen
    Given ein Listener mit facing=0 an Position [0, 0, 0]
    When moveForward(1) aufgerufen wird
    Then ist die neue Position [0, 1, 0]

  Scenario: Seitwärts bewegen
    Given ein Listener mit facing=0
    When strafeRight(1) aufgerufen wird
    Then ist die neue Position [1, 0, 0]

  Scenario: Drehen ändert die Bewegungsrichtung
    Given ein Listener mit facing=90
    When moveForward(1) aufgerufen wird
    Then ist die neue Position [1, 0, 0] (nicht mehr [0, 1, 0])

  Scenario: Tastatursteuerung
    Given ein geöffnetes, fokussiertes Kontrollfenster
    When W/S/A/D/Q/E gehalten wird
    Then bewegt/dreht sich der Listener kontinuierlich, solange die Taste gehalten wird
```

## 3.0 How

### 3.1 Implementation Context

* `classes/sg/spatial/Listener.sc` — vier neue Methoden, alle relativ zu `facing`:
  - `moveForward(d)` / `moveBackward(d)` — entlang des Vorwärtsvektors
    `[sin(facing.degrad), cos(facing.degrad), 0]`
  - `strafeLeft(d)` / `strafeRight(d)` — entlang des Rechtsvektors
    `[cos(facing.degrad), sin(facing.degrad).neg, 0]`
  - `rotate(deltaDegrees)` — `facing = facing + deltaDegrees`
* Neu `classes/sg/spatial/KeyboardListenerControl.sc` — öffnet ein `Window` mit
  `keyDownAction`/`keyUpAction`, hält ein Set gerade gedrückter Tasten, tickt per eigener
  `Routine` (~30Hz) und bewegt/dreht den `Listener` je nach gehaltenen Tasten:
  **W**/**S** vor/zurück, **A**/**D** seitlich, **Q**/**E** drehen.
* Tests: `tests/sg/spatial/TestListener.sc` für die vier Bewegungsmethoden (reine Geometrie,
  inkl. Fall "Rotation ändert Bewegungsrichtung"). `KeyboardListenerControl` selbst (Fenster/
  Tastatur) nicht automatisiert testbar — Verifikation per Ausprobieren.
