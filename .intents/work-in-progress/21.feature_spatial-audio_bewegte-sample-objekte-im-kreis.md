# 21 - Feature (Spatial-Audio): Bewegte Sample-Objekte im Kreis

## 1. Why

### 1.1 Objective

Variante von `sample_demo.scd` (Intent 20): dieselben rhythmischen Sample-Objekte, aber
jedes bewegt sich statt stationär zu bleiben auf einer Kreisbahn — alle mit demselben
Radius und derselben Drehrichtung, aber unterschiedlicher Winkelgeschwindigkeit, und jedes
startet an einer anderen Stelle auf dem Kreis.

### 1.2 Context

`sample_demo.scd` platziert die Objekte per `SteadyMoveRule` fest auf dem Kreis.
`CircularMoveRule` (Intent 9) existiert schon für Kreisbahnen, hat aber kein
"Startwinkel"-Konzept: `next` berechnet den Winkel rein aus `angle = t * angularSpeed` —
jedes Objekt startet bei `t=0` also auf demselben Winkel (0), unabhängig von seiner
`Movable`-Startposition. Für unterschiedliche Startpositionen auf dem Kreis braucht
`CircularMoveRule` deshalb einen zusätzlichen `startAngle`-Parameter.

### 1.3 Depends On Intents

* 9 - Enhancement (Spatial-Audio): Insekt-Architektur aufteilen — liefert `CircularMoveRule`,
  die hier erweitert wird.
* 20 - Feature (Spatial-Audio): Rhythmische Sample-Wiedergabe im Kreis — liefert
  `SampleSound`/das Kreis-Setup, das hier als Vorlage dient.

## 2.0 What

```gherkin
Feature: Bewegte Sample-Objekte im Kreis

  Als Hörer mit Kopfhörern
  möchte ich mehrere Sample-Objekte hören, die sich auf einer gemeinsamen Kreisbahn um mich
  bewegen, statt fest zu stehen
  damit ich rhythmische Sample-Texturen mit Bewegung statt Statik ausprobieren kann.

  Scenario: CircularMoveRule mit Startwinkel
    Given eine CircularMoveRule mit gegebenem startAngle
    When next() bei t=0 aufgerufen wird
    Then liegt die Position bei genau diesem Startwinkel auf dem Kreis, nicht bei Winkel 0

  Scenario: Mehrere Objekte, gleicher Radius, gleiche Richtung, verschiedene Geschwindigkeit
    Given N SoundObjects mit CircularMoveRule (gleicher baseRadius, gleiches Vorzeichen von
      angularSpeed, unterschiedlicher Betrag, gleichmässig verteilter startAngle)
    When sample_demo_moving.scd Block 2 ausgeführt wird
    Then bewegen sich alle Objekte auf derselben Kreisbahn in dieselbe Richtung, aber mit
      unterschiedlichem Tempo — ihre Abstände zueinander verändern sich hörbar über die Zeit
```

## 3.0 How

### 3.1 Implementation Context

* `classes/sg/soundobjects/CircularMoveRule.sc`: neuer Parameter `startAngle = 0` (ans Ende
  der Argumentliste, rückwärtskompatibel), `angle = (t * angularSpeed) + startAngle`.
  `tests/sg/soundobjects/TestCircularMoveRule.sc`: ein Testfall für `startAngle` ergänzen
  (reine sclang-Logik, TDD wie beim Rest der Klasse).
* Neues `sample_demo_moving.scd` (Kopie/Variante von `sample_demo.scd`): identisches Setup,
  aber nur Samples aus `sounds/compmusic/` (nicht der ganze `sounds/`-Baum — auf Wunsch des
  Nutzers, `apple/` bleibt aussen vor), `AtkBinauralizer`, Listener im Zentrum, `SpaceView`;
  `SteadyMoveRule` durch `CircularMoveRule` ersetzt — gleicher
  `baseRadius` für alle, `breathAmount = 0` (Radius bleibt exakt gleich, kein Atmen, wie
  gewünscht), `startAngle` gleichmässig über den Kreis verteilt (`i/n * 2pi`, wie schon für
  die Kreis-Position in Intent 20), `angularSpeed` pro Objekt unterschiedlich, aber alle mit
  gleichem Vorzeichen (gleiche Drehrichtung) — z.B. linear gestaffelt über einen Bereich.
