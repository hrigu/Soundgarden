# 5 - Feature (Spatial-Audio): ATK-basiertes HRTF für das Insekt

## 1. Why

### 1.1 Objective

Die einfache Pan/ITD-Näherung aus Intent 3 durch echtes HRTF-Rendering über das Ambisonic
Toolkit (ATK) ersetzen, damit Vorne/Hinten für das Insekt eindeutig hörbar wird. Scope bewusst
auf die horizontale Ebene beschränkt (siehe 1.4) — Elevation ist ein möglicher Folge-Intent.

### 1.2 Context

Intent 3 lieferte eine funktionierende, aber bewusst einfache Binauralisierung mit dem bekannten
Cone-of-Confusion-Problem. Intent 4 (Research) hat die Optionen verglichen; der Nutzer hat sich
für ATK entschieden — bester Realismus (diffusfeld-equalisierte HRTF-Kernel), trotz des
Installationsaufwands.

### 1.3 Depends On Intents

* 3 - Feature (Spatial-Audio): Binaurales Insekt-Prototyp — liefert `Movable`, `Listener`,
  `SoundObject` und `insect_demo.scd`, die hier erweitert werden.
* 4 - Research (Spatial-Audio): HRTF-Optionen für SuperCollider evaluieren — Entscheidungsgrundlage
  für ATK.

### 1.4 Scope-Anpassung (nachträglich, vor Task 2.0)

Der Intent wurde vor dem Sound/SoundObject/Binauralizer-Refactor (Intents 9–13) geplant und
nennt noch ein `insectVoice`-SynthDef direkt in `insect_demo.scd`, statt der heutigen
Architektur (`Sound`/`InsectSound` → `Binauralizer`). Zusätzlich verspricht das ursprüngliche
Gherkin auch Elevation — das aktuelle Datenmodell ist aber rein 2D (`CircularMoveRule` liefert
immer `z=0`, `Listener` kennt keine Elevation). Mit dem Nutzer abgestimmt: Intent 5 bleibt auf
Azimuth/Distanz (horizontale Ebene) beschränkt, Elevation wird ein eigener späterer Intent, wenn
das Datenmodell dafür erweitert wird.

## 2.0 What

```gherkin
Feature: ATK-basiertes HRTF für das Insekt

  Als Hörer mit Kopfhörern
  möchte ich das Insekt mit echtem HRTF hören
  damit Vorne/Hinten eindeutig unterscheidbar ist, statt mehrdeutig wie bei reinem Pan/ITD.
  (Elevation bewusst außerhalb dieses Intents, siehe 1.4 — Movable liefert aktuell nur z=0.)

  Scenario: ATK ist installiert und verfügbar
    Given eine frische SuperCollider-Installation
    When der ATK-Quark installiert und die Klassenbibliothek neu kompiliert wurde
    Then stehen die ATK-Encoder-/Decoder-Klassen zur Verfügung

  Scenario: Insekt-Position wird nach Ambisonics codiert
    Given Azimuth und Distanz des Insekts relativ zum Listener (aus Orchestra.tick, wie bisher)
    When der Binauralizer-Synth läuft
    Then wird das Mono-Signal per PanB2 auf diese horizontale Position ins B-Format codiert

  Scenario: Binaural-Decoder rendert die Position mit HRTF
    Given ein B-Format-codiertes Signal (W/X/Y)
    When es durch den ATK-HRTF-Decoder (FoaDecoderKernel) läuft
    Then hört man über Kopfhörer eine klar unterscheidbare Vorne/Hinten-Position

  Scenario: Bewegung bleibt klickfrei
    Given das Insekt bewegt sich kontinuierlich (Kreisbahn aus insect_demo.scd)
    When sich Azimuth/Distanz ändern
    Then gibt es keine Klicks oder Aussetzer beim Positionswechsel
```

## 3.0 How

### 3.1 Implementation Context

* ATK-Quark (`ambisonictoolkit/atk-sc3`) installieren, ggf. zusammen mit `sc3-plugins` für
  kompilierte Ambisonics-/Binaural-UGens — genaue Installationsschritte anhand der
  ATK-Dokumentation zum Implementierungszeitpunkt verifizieren.
* Tatsächliche Architektur (siehe 1.4): Signalweg läuft über `Sound`/`InsectSound` (mono, eigener
  Bus) → `Binauralizer` (liest den Bus, schreibt Stereo). `SoundObject`/`Orchestra` bleiben
  unverändert — sie kennen weiterhin nur Azimuth/Distanz, keine ATK-Details.
* Neue Klasse `AtkBinauralizer` (gleiche Schnittstelle wie `Binauralizer`: `play`/`set`/`stop`),
  statt die bestehende Klasse umzubauen — austauschbar über `SoundObject`, alte Pan/ITD-Variante
  bleibt als Fallback unverändert erhalten (wie ursprünglich geplant).
* Encoding: `PanB2.ar(sig, azimuth, distanceAmp)` — Mono → horizontales B-Format (W/X/Y), kr-Rate
  Azimuth-Input ist bereits klickfrei für kontinuierliche Bewegung. Distanzabhängiger Pegel
  weiterhin wie in `Binauralizer` (`1/(1+dist*ampRolloff)`-Formel wiederverwenden).
* Decoding: `FoaDecoderKernel` (CIPIC- oder Spherical-Kernel, je nach Ergebnis des Hörtests) +
  `FoaDecode.ar`, B-Format → Kopfhörer-Stereo.
