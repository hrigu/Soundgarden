# 5 - Feature (Spatial-Audio): ATK-basiertes HRTF für das Insekt

## 1. Why

### 1.1 Objective

Die einfache Pan/ITD-Näherung aus Intent 3 durch echtes HRTF-Rendering über das Ambisonic
Toolkit (ATK) ersetzen, damit Vorne/Hinten und Elevation für das Insekt eindeutig hörbar werden.

### 1.2 Context

Intent 3 lieferte eine funktionierende, aber bewusst einfache Binauralisierung mit dem bekannten
Cone-of-Confusion-Problem. Intent 4 (Research) hat die Optionen verglichen; der Nutzer hat sich
für ATK entschieden — bester Realismus (diffusfeld-equalisierte HRTF-Kernel), trotz des
Installationsaufwands.

### 1.3 Depends On Intents

* 3 - Feature (Spatial-Audio): Binaurales Insekt-Prototyp — liefert `Movable`, `Listener`,
  `SoundObject` und `insect_demo.scd`, die hier erweitert werden.
* 4 - Research (Spatial-Audio): HRTF-Optionen für SuperCollider evaluieren — Entscheidungsgrundlage
  für ATK.

## 2.0 What

```gherkin
Feature: ATK-basiertes HRTF für das Insekt

  Als Hörer mit Kopfhörern
  möchte ich das Insekt mit echtem HRTF hören
  damit Vorne/Hinten und Oben/Unten eindeutig unterscheidbar sind, statt mehrdeutig wie bei
  reinem Pan/ITD.

  Scenario: ATK ist installiert und verfügbar
    Given eine frische SuperCollider-Installation
    When der ATK-Quark installiert und die Klassenbibliothek neu kompiliert wurde
    Then stehen die ATK-Encoder-/Decoder-Klassen zur Verfügung

  Scenario: Insekt-Position wird nach Ambisonics codiert
    Given Azimuth, Elevation und Distanz des Insekts relativ zum Listener
    When der insectVoice-Synth läuft
    Then wird das Signal per Ambisonics-Encoder auf diese Position codiert

  Scenario: Binaural-Decoder rendert die Position mit HRTF
    Given ein Ambisonics-codiertes Signal
    When es durch den ATK-Binaural-Decoder läuft
    Then hört man über Kopfhörer eine klar unterscheidbare Vorne/Hinten- und Höhen-Position

  Scenario: Bewegung bleibt klickfrei
    Given das Insekt bewegt sich kontinuierlich (Kreisbahn aus insect_demo.scd)
    When sich Azimuth/Elevation/Distanz ändern
    Then gibt es keine Klicks oder Aussetzer beim Positionswechsel
```

## 3.0 How

### 3.1 Implementation Context

* ATK-Quark (`ambisonictoolkit/atk-sc3`) installieren, ggf. zusammen mit `sc3-plugins` für
  kompilierte Ambisonics-/Binaural-UGens — genaue Installationsschritte anhand der
  ATK-Dokumentation zum Implementierungszeitpunkt verifizieren.
* `insectVoice`-SynthDef (aus `insect_demo.scd`) um einen Ambisonics-Encoder erweitern
  (Azimuth/Elevation/Distanz als Encoder-Parameter statt direktem Pan/ITD).
* Neuer Decoder-Synth mit ATK-HRTF-Kernel (Kopfform/SubjectID probeweise wählen), B-Format →
  Kopfhörer-Stereo.
* Alte Pan/ITD-Variante aus Intent 3 als Kommentar/Fallback im Code erhalten (z.B. per Schalter),
  falls ATK auf einer Maschine mal fehlt.
