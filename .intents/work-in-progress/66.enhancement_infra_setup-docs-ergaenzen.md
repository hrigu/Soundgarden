# 66 - Enhancement (Infra): Setup-Dokumentation in CLAUDE.md und AGENTS.md ergänzen

## 1. Why

### 1.1 Objective
Zwei kleine Lücken in der Setup-Dokumentation schliessen:

1. `.claude/settings.local.json` ist zwar vorhanden, wird aber in `CLAUDE.md` nicht erwähnt.
   Ein neuer Agent weiss deshalb nicht, dass dort zusätzliche Berechtigungen für
   `~/Library/Application Support/SuperCollider/**` und Git-Operationen definiert sind.
2. `AGENTS.md` listet nur drei Domains (`Live-Set`, `Boot-Tracks`, `Spatial-Audio`), obwohl
   im Projekt längst weitere etabliert sind (`infra`, `sounds`, `soundobjects`, `spatial`,
   `gui`). Das führt zu Inkonsistenzen in Commit-Messages und Intent-Dateinamen.

### 1.2 Context
Beide Dateien sind die zentralen Einstiegspunkte für Agenten. Je präziser sie den aktuellen
Projektzustand abbilden, desto weniger Interpretationsspielraum bleibt.

### 1.3 Related To Intents
- 63 - Headless-Test-Skript
- 64 - Parkierte Intents
- 65 - Gotcha-Index in CLAUDE.md

## 2.0 What

```gherkin
Feature: Setup-Dokumentation ergänzen

  Scenario: Agent findet .claude-Einstellungen
    Given ein neuer Agent liest CLAUDE.md
    When er den Bereich "Befehle" oder "Agent-spezifische Einstellungen" erreicht
    Then sieht er einen Hinweis auf `.claude/settings.local.json`

  Scenario: Intent-Domain passt zur Projekt-Realität
    Given ein Agent schreibt eine Commit-Message oder einen Intent
    When er die Domain wählt
    Then stehen ihm neben den drei Kern-Domains auch die etablierten Unter-Domains zur Verfügung
```

## 3.0 How

### 3.1 Implementation Context
- `CLAUDE.md`: neuer kurzer Hinweis auf `.claude/settings.local.json`.
- `AGENTS.md`: Abschnitt "Intent Driven Development (IDD)" → "Domains" erweitern.
