# 65 - Enhancement (Infra): Gotcha-Index in CLAUDE.md

## 1. Why

### 1.1 Objective
Die Fallstricke in `CLAUDE.md` sind wertvoll, aber in einem langen Fließtext vergraben. Ein
kurzer Index am Anfang der Gotchas-Sektion macht sie auffindbarer, wenn man an einer
bestimmten Klasse oder einem bestimmten Problem arbeitet.

### 1.2 Context
`CLAUDE.md` listet unter „Gotchas“ mehrere SuperCollider-spezifische Fallstricke
(`PathName>>extension`, `rrand(*array)`, Bluetooth-Headset-Samplerate,
`CricketSound(pattern: nil)`, etc.). Diese werden in Commit-Messages und Code-Kommentaren
referenziert, aber ohne Index muss man den gesamten Abschnitt lesen, um das Richtige zu finden.

### 1.3 Related To Intents
- 32 - load-classes Doku pro Sitzung
- 61 - Grillenchor mit emergenter Synchronisation (CricketSound-Pattern-Fallstrick)

## 2.0 What

```gherkin
Feature: Gotcha-Index in CLAUDE.md

  Scenario: Agent sucht einen bekannten Fallstrick
    Given ich arbeite an `BootTrackDetection`
    When ich die Gotchas-Sektion öffne
    Then sehe ich sofort, dass es einen Eintrag zu Extension-Vergleichen gibt
    And kann direkt zur Erklärung springen

  Scenario: Neue Gotchas lassen sich indexieren
    Given ein neuer Fallstrick wird in CLAUDE.md ergänzt
    When die Sektion wächst
    Then wird auch der Index um eine Zeile erweitert
```
