# 67 - Bug (Spatial-Audio): reverbMix-Warnung in TestListener beheben

## 1. Why

### 1.1 Objective
Beim Ausführen der Unit-Tests erscheint im Kontext von `TestListener` eine Warnung:

```
WARNING: keyword arg 'reverbMix' not found in call to Meta_Object:new
```

Diese Warnung verursacht zwar keinen Test-Fail, deutet aber auf einen leichten API-Drift oder
einen unbeabsichtigten Konstruktor-Aufruf hin. Sie sollte bereinigt werden, um zukünftige
Fehlalarme zu vermeiden und die Testausgabe sauber zu halten.

### 1.2 Context
Die Warnung taucht in `TestListener` auf, vermutlich in Verbindung mit dem Fallback auf einen
`directOutBinauralizerClass`, wenn `binauralizerClass = nil` gesetzt wird. Möglicherweise wird
`reverbMix` an eine Klasse übergeben, die dieses Keyword-Argument nicht im `*new`-Methoden-
kopf definiert hat.

### 1.3 Related To Intents
- 27 - Binauralizer als Interna des Listeners
- 51 - Room konfiguriert Binauralizer zentral
- 52 - Binauralizer-Verantwortung zurück zum Listener

## 2.0 What

```gherkin
Feature: reverbMix-Warnung in TestListener entfernen

  Scenario: Tests laufen ohne Warnung
    Given ich führe `run_tests.scd` aus
    When die Tests für `TestListener` ausgeführt werden
    Then erscheint keine Warnung zu `reverbMix` und `Meta_Object:new`

  Scenario: Verhalten bleibt erhalten
    Given ein Listener mit binauralizerClass = nil
    When setup ausgeführt wird
    Then wird weiterhin der DirectOut-Fallback verwendet
    And reverbMix wird korrekt an den Fallback weitergegeben
```

- `classes/sg/spatial/DirectOutBinauralizer.sc` — Fallback-Klasse.
