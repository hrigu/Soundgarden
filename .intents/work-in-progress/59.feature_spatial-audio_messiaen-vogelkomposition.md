# 59 - Feature (Spatial-Audio): Messiaen-inspirierte Vogel-Timeline-Komposition

## 1. Why

### 1.1 Objective

Olivier Messiaen transkribierte Vogelgesänge in Musiknotation (u.a. *Catalogue d'oiseaux*) und
nannte Vögel "die grössten Musiker der Erde". Der Nutzer möchte ein eigenständiges Klangobjekt
bauen, das solche Vogel-Motive — melodische Tonhöhenfolgen statt reiner An/Aus-Rufrhythmen —
automatisch über eine Timeline-Komposition abspielt, räumlich verteilt wie die bestehende
Dämmerungs-Komposition (Intent 57/58).

### 1.2 Context

Kein Live-Spiel-Interface (Keyboard/MIDI) nötig — Nutzer-Entscheid: automatische Komposition
statt spielbares Instrument. Keine echten Catalogue-d'oiseaux-Zitate: Messiaens Transkriptionen
sind urheberrechtlich geschützt (Messiaen † 1992, Schutzfrist in EU/CH bis ~2062). Stattdessen
werden eigene, Messiaen-inspirierte Motive komponiert (charakteristische Merkmale: perlende
schnelle Läufe, Triller-Kaskaden, große Intervallsprünge) — keine exakte Transkription.

Bestehende Infrastruktur (`classes/sg/`) bietet Timeline-Komposition, Sound-Basisklasse-
Konventionen und Raum/Binauralisierung — aber keine Tonhöhenfolge-/Melodie-Infrastruktur:
`BirdSound` (Intent 57) hat nur eine feste `baseFreq`, `CallingPattern`/`RhythmPatternCreator`
sind reine An/Aus-Rhythmus-Iteratoren ohne Tonhöhe. Diese Lücke ist der Kern der neuen Arbeit.
Nutzer-Entscheid: neue eigenständige Klangart statt Ausbau von `BirdSound`/`SampleSound`.

Depends On: Intent 57/58 (Timeline-Modell, `fadeOutAndStop`, `staggeredTimes`,
`oeuvre/dawn/dawn_chorus.scd` als Strukturvorbild).

## 2.0 What

```gherkin
Feature: Messiaen-inspirierte Vogel-Timeline-Komposition

  Scenario: Motiv als Tonhöhen-/Rhythmus-Sequenz
    Given eine Liste von Halbtonschritten, eine Startfrequenz und Notendauern
    When BirdMotif.fromIntervals aufgerufen wird
    Then liefert es ein Motiv mit korrekt in Hz umgerechneten Tonhöhen
    And totalDur/noteEvents liefern korrekte kumulative Zeitwerte

  Scenario: Beispiel-Motive verfügbar
    Given BirdMotifExamples
    When blackbirdRun/wrenTrillCascade/callBirdLeaps aufgerufen werden
    Then liefert jede Fabrik ein gültiges BirdMotif mit plausibler Notenanzahl

  Scenario: Ein Motiv klingt als Vogel
    Given ein MessiaenBirdSound mit einem Motiv
    When play aufgerufen wird
    Then durchläuft eine Routine die Noten des Motivs und retriggert pro Note
      Tonhöhe (freq) und Hüllkurve (t_trig) auf einem einzigen laufenden Synth

  Scenario: Komposition klingt räumlich verteilt
    Given oeuvre/messiaen_birds/messiaen_birds.scd
    When das Stück mit Kopfhörern abgespielt wird
    Then setzen mehrere Motiv-Typen zeitversetzt/überlappend ein, räumlich
      verteilt im Raum (Room/Listener/AtkBinauralizer)
    And beim Stück-Ende verstummen alle Vögel klickfrei (fadeOutAndStop)
```

## 3.0 How

### 3.1 Implementation Context

- `classes/sg/sounds/BirdMotif.sc` (neu): reine, server-freie Datenstruktur (analog
  `CallingPattern`). Noten als `[freq, dur, gateRatio]`, `*fromIntervals` für
  Halbtonschritt→Hz-Umrechnung, `totalDur`/`noteEvents` für Zeitberechnung.
- `classes/sg/sounds/BirdMotifExamples.sc` (neu): 3 Platzhalter-Motiv-Fabriken
  (`blackbirdRun`, `wrenTrillCascade`, `callBirdLeaps`), getrennt von `BirdMotif` selbst.
- `classes/sg/sounds/MessiaenBirdSound.sc` (neu): erbt von `Sound` (`classes/sg/sounds/Sound.sc`),
  Motiv als Pflichtargument (`*requiredConstructorArgs`, wie `path` bei `SampleSound`), Routine
  treibt `freq`/`noteDur`/`t_trig` auf einem laufenden Synth — Muster wie
  `BirdSound>>play`/`patternRoutine`. SynthDef `\messiaenBird`: `Env.perc` pro `t_trig`,
  Grundton + 2. Harmonischer, gefilterter Rausch-"Chiff" am Attack (`brightness`-gesteuert).
- `demos/sounds/messiaenbird.scd` (neu, analog `demos/sounds/insectsound.scd`): Mono-Hörtest-
  Harness für die drei Beispiel-Motive, vor der vollen räumlichen Komposition.
- `oeuvre/messiaen_birds/messiaen_birds.scd` (neu): Strukturvorbild 1:1
  `oeuvre/dawn/dawn_chorus.scd` — `Room`+`Listener`(`AtkBinauralizer`)+`Orchestra`,
  `Timeline.new`, `Timeline.staggeredTimes` pro Motiv-Typ, `~room.registerFromBuilder(...,
  soundClass: MessiaenBirdSound)`, Aufräumen über `SoundObject>>fadeOutAndStop`.
- Tests: `tests/sg/sounds/TestBirdMotif.sc`, `tests/sg/sounds/TestMessiaenBirdSound.sc`
  (Vorlage: `tests/sg/sounds/TestBirdSound.sc`), in `run_tests.scd` ergänzt.
