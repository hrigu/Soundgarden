# 17 - Feature (Spatial-Audio): Soundobjekte rufen sich gegenseitig zu

## 1. Why

### 1.1 Objective

Soundobjekte sollen sich per Live-Coding gegenseitig "zurufen" können — eine kurze,
hörbare Variation des eigenen Klangs, auf die ein anderes registriertes Objekt nach kurzer
Verzögerung antwortet.

### 1.2 Context

Bisher erzeugt jedes Soundobjekt nur seinen kontinuierlichen Grundklang, unabhängig von den
anderen. Interaktion zwischen den Objekten (Call-and-Response) fehlt bisher komplett.

### 1.3 Depends On Intents

* 11 - Enhancement (Spatial-Audio): Sound-Superklasse einführen — liefert `Sound`/`InsectSound`.
* 12 - Enhancement (Spatial-Audio): Orchestra-Klasse einführen — liefert die Registry, die für
  die Partnerwahl gebraucht wird.

## 2.0 What

```gherkin
Feature: Call-and-Response zwischen Soundobjekten

  Als Nutzer, der live codet
  möchte ich ein Soundobjekt "rufen" lassen können
  damit es sich anfühlt, als würden sich die Objekte gegenseitig zurufen.

  Scenario: Ein Objekt ruft
    Given ein SoundObject mit laufendem Sound
    When call() aufgerufen wird
    Then hört man eine kurze Klangvariation (Hüllkurven-Akzent) auf dem laufenden Klang

  Scenario: Ein anderes Objekt antwortet
    Given mehrere bei Orchestra registrierte SoundObjects
    When orchestra.call(einesDavon) aufgerufen wird
    Then ruft nach kurzer, leicht zufälliger Verzögerung ein anderes (nicht dasselbe)
      registriertes Objekt zurück

  Scenario: Nur ein Objekt registriert
    Given genau ein SoundObject bei Orchestra registriert
    When orchestra.call(diesesObjekt) aufgerufen wird
    Then löst nur der ursprüngliche Ruf aus, kein Rückruf (kein anderes Objekt vorhanden)
```

## 3.0 How

### 3.1 Implementation Context

* `classes/sg/sounds/Sound.sc` — neue Methode `call`: `synth.set(\t_call, 1)` (generischer
  Hook; wirkt als No-Op, falls die konkrete SynthDef keinen `t_call`-Control besitzt).
* `classes/sg/sounds/InsectSound.sc` — SynthDef um `t_call`-Trigger-Control und
  `EnvGen.kr(Env.perc(...))` erweitert, der Pegel/Klangfarbe kurz "aufblitzen" lässt — keine
  neue SynthDef, nur eine Erweiterung der bestehenden.
* `classes/sg/soundobjects/SoundObject.sc` — neue Methode `call`, delegiert an `sound.call`.
* `classes/sg/spatial/Orchestra.sc` — neue Methode `call(caller)`: löst `caller.call` aus und
  wählt ein zufälliges **anderes** registriertes Objekt (Auswahllogik als eigene, pure Methode
  ausgelagert und separat testbar — analog zu `tick`), das nach kurzer, leicht zufälliger
  Verzögerung (`Routine` + `.wait`) selbst `call` aufruft. Bewusst bei `Orchestra`, nicht bei
  `SoundObject` — nur `Orchestra` kennt die Registry aller Objekte (Prinzip aus Intent 12).
* Aufruf im Live-Set: `~orchestra.call(~insect1)`.
* Tests: `tests/sg/spatial/TestOrchestra.sc` erweitert um die Auswahllogik (nie der Aufrufer
  selbst, `nil`/kein Rückruf falls keine anderen Objekte registriert sind) mit Fake-SoundObjects.
