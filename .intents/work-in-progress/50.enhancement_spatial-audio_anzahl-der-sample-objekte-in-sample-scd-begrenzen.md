# 50 - Enhancement (Spatial-Audio): Anzahl der Sample-Objekte in sample.scd begrenzen

## 1. Why

### 1.1 Objective
`demos/sample.scd` so anpassen, dass aus dem Percussion-Ordner nicht mehr für jede Datei ein
eigenes Soundobjekt erzeugt wird, sondern höchstens `n` Objekte.

### 1.2 Context
Im Ordner `sounds/instruments/Percussion` liegen aktuell sehr viele Dateien (ca. 148). Wenn
`sample.scd` daraus für jede Datei ein eigenes Soundobjekt baut, wird das Demo unnötig dicht und
schwer hörbar. Gewünscht ist eine einfache Begrenzung über eine Variable, zunächst mit `n = 40`.

### 1.3 Related To Intents

* 49 - Preset-Workflow in Song- und Sample-Demos konsistent aktivieren

## 2.0 What

```gherkin
Feature: Begrenzte Anzahl von Sample-Objekten im sample-Demo

  Scenario: Nur höchstens n Samples werden verwendet
    Given ein Ordner mit mehr als n gültigen Audiodateien
    When sample.scd die Sample-Objekte aufbaut
    Then werden höchstens n Soundobjekte erstellt

  Scenario: Weniger Dateien als n
    Given ein Ordner mit weniger als n gültigen Audiodateien
    When sample.scd die Sample-Objekte aufbaut
    Then werden alle verfügbaren Dateien verwendet

  Scenario: Anzahl ist im Demo leicht anpassbar
    Given ich möchte dichter oder sparsamer testen
    When ich die Variable n ändere
    Then ändert sich die maximale Zahl der erzeugten Soundobjekte entsprechend
```

## 3.0 How

### 3.1 Implementation Context

* Betroffen ist nur `demos/sample.scd`
* Nach dem Einsammeln der gültigen Dateien wird die Liste auf höchstens `n` Einträge begrenzt
* `n` soll im Setup-Block als klar sichtbare Variable stehen, initial `40`
* Keine Änderung an `SampleSound`, `Room` oder GUI-Logik nötig

- [ ] 1.0 Maximale Sample-Anzahl als Variable in `sample.scd` einführen
- [ ] 2.0 Dateiliste auf höchstens `n` Einträge begrenzen
- [ ] 3.0 Demo-Kommentar/Output kurz anpassen
- [ ] 4.0 Relevante Tests laufen lassen und Intent abschliessen
